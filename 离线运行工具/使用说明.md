# 离线单文件 HTML 摄像头问题 · 一键解决工具

## 问题是什么

双击 HTML 文件时，浏览器是通过 `file://` 协议打开的。浏览器安全策略规定：
**摄像头（`getUserMedia`）只能在"安全上下文"中使用**，即 `https://` 页面或
`http://localhost`。`file://` 不属于安全上下文，所以「量子迷宫」手势识别页面
会一直显示"未授权或无可用摄像头"，无论怎么点允许都没用。

解决办法就是别用 `file://` 打开，改为通过本地 HTTP 服务器用
`http://localhost:8000/...` 打开。**服务器只跑在你自己电脑上，全程不需要联网。**

## 使用步骤（Windows）

1. 把本文件夹里的两个文件——`启动本地服务器.bat` 和 `server.ps1`——
   复制到离线 HTML 所在的文件夹（例如
   `D:\College\IndustrialDesign\Term 4\UI设计\科创文旅\1\`）。
2. 双击 `启动本地服务器.bat`。
3. 浏览器会自动打开 `http://localhost:8000/`，显示该文件夹的文件列表；
   点击 `量子探微馆-离线单文件-最终更新.html` 打开页面。
4. 进入「展区总览 → 引领区 → 量子迷宫」，浏览器会弹出摄像头授权请求，
   点"允许"即可开始手势交互。
5. 体验结束后直接关闭黑色命令行窗口，服务器就停止了。

> 脚本会自动选择启动方式：装了 Python 就用 `python -m http.server`；
> 没装 Python 也没关系，会自动改用 Windows 自带的 PowerShell 启动
> 一个内置的小型服务器（`server.ps1`），无需安装任何软件。

## 常见问题

- **提示端口被占用 / 页面打不开**：用记事本打开 `启动本地服务器.bat`，
  把 `set PORT=8000` 改成 `set PORT=8010`，保存后重新双击运行，
  然后访问 `http://localhost:8010/`。
- **仍然提示无摄像头**：确认地址栏是 `http://localhost:...` 开头而不是
  `file:///...`；再检查系统设置里是否允许浏览器使用摄像头
  （Windows 设置 → 隐私和安全性 → 相机）。
- **想在 iPad / 其它设备上打开**：`localhost` 只对本机有效。其它设备通过
  局域网 IP（如 `http://192.168.x.x:8000`）访问时不属于安全上下文，
  摄像头仍会被浏览器拒绝，需要配置 HTTPS 证书才行，属于另一套方案。

## 手动方式（不用脚本）

在 HTML 所在文件夹打开终端，任选其一：

```bash
# 装了 Python
python -m http.server 8000

# 装了 Node.js（首次运行需联网下载 http-server）
npx http-server -p 8000
```

然后浏览器访问 `http://localhost:8000/`，点击对应的 HTML 文件。
